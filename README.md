# Data Engineering Interview Cheatsheets

A self-contained set of fast-revision cheatsheets for **Senior / Lead Data Engineering**
interviews. Concepts, diagrams, gotchas, "interviewer will probe" callouts, and flip-card
self-quizzes.

Every page is:
- **Bilingual** — switch between English and Russian with the 🌐 button (choice is remembered).
- **Themed** — dark / light toggle with the 🌙 button.
- **Offline-first** — no CDN, no external fonts, no tracking. Just open the HTML.

👉 **Open [`index.html`](index.html)** for the full hub.

## Topics

| Area | Cheatsheets |
|------|-------------|
| Streaming & messaging | Kafka, Flink |
| Batch compute & language | Spark, Scala, Coding & Algorithms, Python & SQL |
| Storage & modeling | Iceberg & Warehousing, SQL & Data Modeling, Modeling / Governance / Lineage |
| Pipelines & platform | Airflow, Pipeline Design, Cloud / DataOps / CI-CD, Legacy Modernization |
| Quality, testing & ML | Software Testing for DE, Data Quality & Observability, ML/AI for Data Quality |
| System design & domain | Data Platform Design, Economics & Time-Series Domain |
| Behavioral & collaboration | Behavioral & Values, Behavioral / Mentorship / Values, Stakeholders & Requirements |

A dedicated multi-level **Distributed Systems** project (beginner → intermediate → advanced)
is in the works and will be linked from the hub.

## How to use

1. Read the concepts top-to-bottom.
2. Study the **"interviewer will probe"** callouts — these are the follow-up questions.
3. Flip the **self-quiz** cards and answer out loud *before* revealing.
